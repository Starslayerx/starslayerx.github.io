本篇博客记录一下学习 PostgreSQL 过程中的一些概念、语法等内容。

## 表

### 创建表

创建表的语法如下

```sql
CREATE TABLE table_name(
    col01_name data_type,
    col02_name data_type,
    col03_name data_type,
    col04_name data_type,
);
```

一般表都会有主键，如果主键由单个字段组成，则直接在后面添加 `PRIMARY KEY` 关键字指定：

```sql
CREATE TABLE test_table01(id int PRIMARY KEY, note varchar(20));
```

如果主键由两个及以上的字段组成，则需要使用约束子句的语法，指定符合主键的约束语法如下：

```sql
CONSTRAINT constraint_name PRIMARY KEY (col1_name, col2_name, ...)
```

例如：

```sql
CREATE TABLE test02(
    id1 int, id2 int, note varchar(20),
    CONSTRAINT pk_test02 PRIMARY KEY(id1, id2)
);
```

约束是放在定义后面的，建表的使用也可以指定唯一键：

```sql
CONSTRAINT constraint_name UNIQUE(col1_name, col2_name, ...)
```

例如：

```sql
CREATE TABLE test03(
id1 int, id2 int, id3 int, note varchar(20),
CONSTRAINT uk_test03_id3 UNIQUE(id3)
);
```

此外，check 也是一种约束，用于定义某些字段的值必须满足某种要求，语法如下：

```sql
CONSTRAINT constraint_name CHECK(expression)
```

例如，创建一张孩子表 child，要求年龄字段 age 不能大于 18 岁：

```sql
CREATE TABLE child(
    name varchar(20), age int, note text,
    CONSTRAINT ck_child_age CHECK(age < 18)
);
```

还可以使用其他表作为模板来创建表：

```sql
CREATE TABLE baby (LIKE child);
```

但要注意，新建的 baby 表并不会将源列表的约束复制过来。
如果要复制完整信息，需要加上 `INCLUDING` 关键字：

- INCLUDING DEFAULTS
- INCLUDING CONSTRAINTS
- INCLUDING INDEXES
- INCLUDING STORAGE
- INCLUDING COMMENTS
- INCLUDING ALL

例如：

```sql
CREATE TABLE baby2 (LIKE child INCLUDING ALL);
```

当然也可以使用 `CREATE TABLE ... AS` 来创建表，如下：

```sql
CREATE TABLE baby2 AS SELECT * FROM child WITH NO DATA;
```

### 临时表

PostgreSQL 有两种临时表：

- 会话级别临时表：数据可以一直保存在整个会话的生命周期中
- 事务级别临时表：数据只存在于该事务的生命周期中

使用下面语法创建一张临时表

```sql
create TEMPORARY table tmp_t1(id int PRIMARY KEY, note text);
```

默认创建的临时表是会话级的，如果想创建事务级的表，可以加 `ON COMMIT DELETE ROWS` 子句，实例如下：

```sql
creaet TEMPORARY table tmp_t2(id int PRIMARY KEY, note text) ON COMMIT DELETE ROWS;
```

关键临时表的关键字 `TEMPORARY` 可以缩写为 `TEMP`，下面两条 SQL 语句是等价的：

```sql
create TEMPORARY table tmp_t1(id int PRIMARY KEY, note text);
create TEMP table tmp_t1(id int PRIMARY KEY, note text);
```

为了和其他数据库兼容，PostgreSQL 还可以使用 `GLOBAL` 和 `LOCAL` 两个关键字。
但这两个关键字没有任何作用，下面的几条 SQL 语句都是等价的：

```sql
create TEMPORARY table tmp_t1(id int PRIMARY KEY, note text);
create GLOBAL TEMPORARY table temp_t1(id int PRIMARY KEY, note text);
create LOCAL TEMPORARY table temp_t1(id int PRIMARY KEY, note text);
```

### UNLOGGED 表

`UNLOGGED` 表是 pg 9.1 版本开始新增的一种表，主要是通过禁止产生 WAL 日志的方式提升读写性能。
因为没有 WAL 日志，所以表的内容无法在主备库直接同步。
如果此时数据库异常宕机，表的内容将丢失，所以可以把 `UNLOGGED` 表称为 “半临时表”。
如果数据库是正常关机的，则 `UNLOGGED` 表的内容不会丢失。

创建 `UNLOGGED` 表命令如下：

```sql
CREATE UNLOGGED TABLE unlogged01(id int PRIMARY KEY, t text);
```

`UNLOGGED` 表和普通表没有区别，仅仅在插入、删除、更新数据时不产生 WAL 日志，所以做这些 DML 操作的性能会更高。
要注意的是，数据库异常宕机时，`UNLOGGED` 表的数据可能丢失。

### 默认值

建表时可以为字段指定默认值，对于指定默认值的列，如果插入了新行，但没有声明该字段，则会使用默认值填充。

使用 `DEFAULT` 关键字来指定默认值：

```sql
CREATE TABLE student(no int, name varchar(20), age int DEFAULT 15);
```

使用 `UPDATE` 语句的时候，也可以使用 `DEFAULT` 来代表默认值：

```sql
update student set age=16;
update student set age=DEFAULT where no=2;
```

如果没有默认值，那么默认是 NULL，因为 NULL 表示未知。

默认值可以是一个表达式，在插入默认值的时候进行计算。
一个常见的实例是 `timestamp` 字段可能有默认值 `now()`，它表示插入的时间：

```sql
create table blog(id int, title text, created_date timestamp default now());
```

### 约束

目前，约束主要有以下几类：

- 检查约束
- 非空约束
- 唯一约束
- 主键
- 外键约束

---

1. 检查约束

检查约束是最常见的约束类型。
使用该约束，设置某个字段的数值时，必须使用该约束的表达式值为真。

```sql
CREATE TABLE persons (
    name varchar(40),
    age int CHECK (age > 0 and age <= 150),
    sex boolean
);
```

还可以给约束取一个独立的名称，这样出错的时候就会将约束也打印出来

```sql
CREATE TABLE persons (
    name varchar(40),
    age int CONSTRAINT check_age CHECK
    (age >= 0 and age <= 150),
    sex boolean
);
```

一个检查约束可以引用多个字段。

假设在一张存储图书的表 book 中存储了一个正价和一个打折价，要求打折价比正价低：

```sql
CREATE TABLE books (
    book_no integer,
    name text,
    price numberic CHECK (price > 0),
    discounted_price numberic CHECK (discounted_price > 0),
    CHECK (price > discounted_price)
);
```

前面两个字段和前面的一样，第三个约束使用了一个新的语法，它没有依附在某个字段上，而是在逗号分隔的列表中以一个独立的行出现。

一般称前两个为 “字段约束”，第三个约束为 “表约束”。
大多数数据库中，字段约束也可以写成表约束，但反过来可能不行。

```sql
CREATE TABLE blocks (
    book_no integer,
    name text,
    price numberic,
    discounted_price numberic,
    CHECK (price > 0) CHECK (discounted_price > 0),
    CHECK (price > discounted_price)
);
```



2. 非空约束

非空约束只是简单地声明一个字段必须不能是 NULL，如下：

```sql
CREATE TABLE books (
    book_no integer not null,
    name text,
    price numberic
);
```
